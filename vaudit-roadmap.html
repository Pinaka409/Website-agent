3 · Phase 1 — foundation & first signals

Bring in the first wave of APIs, surface what the agent already detects today, and start logging what the agent is actually saying so accuracy becomes measurable.

Phase 1 Foundation & first signals ~ May 6 target

Three slices ship together. The accuracy track adds six signal APIs (specific picks pending marketing recommendation). The behaviour track strips the "estimate" language and surfaces the sizing detection. The reporting track adds the source receipt line so users can see what the agent is anchored on, and starts logging estimates so we have the data to learn from.

Accuracy Six signal APIs integrated

Specific picks pending Thursday's marketing review. Working assumption is four free public sources + two self-serve paid sources, with the strongest candidates per category listed below.

  • →Google Ads Transparency Centerfree — confirms domain is actively running Google Ads. Binary gate for ADID.
  • →Meta Ad Libraryfree — confirms Meta advertising activity. Same gating role for Meta spend in ADID.
  • →Carrier rate cardsfree — UPS / FedEx / DHL public rates. Wired into SHIPID baseline.
  • →Cloud pricing APIsfree — AWS / GCP / Azure unit pricing. Wired into KLOUDID.
  • →Firmographics providerpending pick — Apollo or PDL. Headcount, revenue band, vertical confirmation. Marketing data quality test will pick the winner.
  • →Traffic baseline providerpending pick — SEMrush Trends Basic is the lead candidate. Marketing's coverage testing across our verticals decides.
Agent behaviour Validation language & sizing readout

The agent's voice changes to match the live site's "we validate, not estimate" positioning, and the sizing detection that runs invisibly today gets surfaced in the response.

  • →Validation language audit. Strip "estimate," "AI-generated," "inferred from benchmarks" from every agent response. Replace with "based on patterns across similar accounts."
  • →Sizing readout one-liner. Above the pitch: "Detected: DTC apparel · 10–50 employees · NA · Shopify + Klaviyo + Google Ads." The work happens today; surfacing it earns credibility.
  • →Range-based numbers. Replace point estimates with low–high bands in agent copy. Annual + monthly side-by-side at the size appropriate to detected company.
Reporting Source receipts & basic logging

User-facing reporting on what the agent is anchored on, plus the first internal logging so accuracy can be measured later.

  • →Source receipt line per ID. "Based on N similar audited accounts." N pulled from internal data. The receipt is the visible reporting layer for the user.
  • →Pre-signup estimate logging. Capture the estimate produced for every domain — vertical, size, range, source N. Foundation for Phase 2's estimate-vs-actual loop.
  • →Stack & signal coverage flag. Which APIs returned data for this domain, which were null. Internal only — feeds debugging and Phase 3 prioritization.
What changes for the user

The agent now shows what it detected before showing the number, the number is a range with credibility line below, and the language matches the rest of the site's positioning. Six new signal sources are working behind the scenes. Internally, every signup is now logged in a way that lets us measure accuracy in Phase 2.

4 · Phase 2 — refinement & learning loop

Add one more signal API, ship the refinement chips on the front-end, and close the first feedback loop with an estimate-vs-actual dashboard.

Phase 2 Refinement & learning loop ~ May 12 target

Phase 1 establishes the foundation. Phase 2 makes it learn. The accuracy track adds one direct ad-spend signal — likely SpyFu, marketing's pick. The behaviour track ships refinement chips and the conversational rules for handling user input. The reporting track lights up the dashboard that compares pre-signup estimates against post-connect actuals.

Accuracy Direct ad-spend signal

One more API on top of Phase 1's six. Working candidate is SpyFu — direct dollar reading on Google Ads spend for US/UK domains. Marketing's coverage check decides whether it ships or gets swapped.

  • →SpyFu APIpending pick — direct estimated Google Ads monthly spend, top keywords, history. Coverage US/UK only and Google-only. Use as ADID enhancer, not primary.
Agent behaviour Refinement chips & override handling

The conversational layer for sharpening estimates. Single-tap chip inputs, live recalibration, and the rules for what the agent says when user input contradicts inference.

  • →Refinement chips UI. Single-tap inputs below the pitch — monthly ad spend, team size, vertical confirm, region. Live recalibration of the bolded number. Signup form stays mounted; chips always optional.
  • →Cohort-aware copy. When chip taps shift the cohort, the source line updates: "Based on 47 similar accounts" → "Based on 12 at your reported scale."
  • →Big-disagreement soft surface. If user input is wildly inconsistent with inference, agent acknowledges with one line: "That's significantly above what we usually see for your stack — connecting your account will give you the exact number."
  • →No follow-up questions rule. Chips are the entire input surface. Agent never asks free-text questions. Flow remains one-shot.
Reporting Estimate-vs-actual dashboard

First closed feedback loop. Every connection a user makes after signup produces a comparison datapoint that can be stacked against the pre-signup estimate.

  • →Estimate-vs-actual pipeline. Match Phase 1's logged estimates against post-connect actuals per ID category. Output: error band per vertical, N-per-vertical.
  • →Internal accuracy dashboard. Ratio band, vertical breakdown, drill-down to specific domains. Owned by data; visible to product and marketing.
  • →Override telemetry. Log every chip tap. User-supplied value, original inferred value, dimension. Three-way comparison feeds the calibration loop.
What changes for the user

Users can now sharpen the estimate with one-tap chips and watch the number recalibrate live. Behind the scenes, accuracy is measurable for the first time — every signup contributes a datapoint. ADID gets a direct dollar reading on US/UK domains where the Phase 2 API has coverage.
